# 团队共享 Skill

## 描述

这是一个用于团队协作的 Skill，支持多用户场景、配置管理和版本控制。

## 用途

为团队提供统一的文档处理、数据分析和报告生成能力。

## 版本信息

- **版本：** 1.0.0
- **最后更新：** 2024-01-01
- **维护者：** 团队开发组

## 配置管理

### 环境变量

使用环境变量管理敏感信息：
- `API_KEY`: API 密钥
- `DATABASE_URL`: 数据库连接字符串
- `LOG_LEVEL`: 日志级别

### 配置文件

读取 `resources/config.json` 获取应用配置。

## 指令

### 文档处理

当用户要求处理文档时：
1. 检查用户权限（读取 `resources/permissions.json`）
2. 调用相应的处理脚本
3. 记录操作日志
4. 返回处理结果

### 数据分析

当用户要求分析数据时：
1. 验证数据文件路径
2. 调用数据分析脚本
3. 生成分析报告
4. 保存结果到共享目录

### 报告生成

当用户要求生成报告时：
1. 读取报告模板（`resources/templates/report_template.md`）
2. 收集数据
3. 填充模板
4. 保存到团队共享目录

## 权限管理

- 读取权限配置：`resources/permissions.json`
- 支持角色：`admin`、`editor`、`viewer`
- 权限检查：在执行操作前验证用户权限

## 日志记录

所有操作都会记录到日志文件：
- 日志位置：`logs/team_skill.log`
- 日志格式：JSON
- 包含信息：用户、操作、时间、结果

## 示例

**用户输入：** "处理文档 /shared/documents/report.docx"

**处理流程：**
1. 检查用户权限
2. 调用文档处理脚本
3. 记录操作日志
4. 返回处理结果

**用户输入：** "生成月度报告"

**处理流程：**
1. 收集本月数据
2. 读取报告模板
3. 填充数据
4. 保存到共享目录
5. 通知团队成员
