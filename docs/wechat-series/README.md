# Skills 公众号系列：从入门到实战与资源库运营

关键词：cursor, claude code, skills, skill

Git 链接：
- 仓库：[shinjiyu/skills_all_in_one](https://github.com/shinjiyu/skills_all_in_one)
- 本文：[docs/wechat-series/README.md](https://github.com/shinjiyu/skills_all_in_one/blob/main/docs/wechat-series/README.md)

这是一套面向公众号读者的 **Skills 系列文章**：按“概念 → 规范 → 写法 → 集成 → 实战 → 团队 → 生产 → 运营”的主线线性展开，并且每篇末尾提供 1 道**判断题**，欢迎读者在评论区回复“对/错”。

## 阅读顺序（12 篇）

1. **01 - Skills 到底是什么：它解决的是“稳定复用”，不是“更长提示词”**
2. **02 - Skills 从哪里来：把高频工作流固化成能力（而不是堆技巧）**
3. **03 - SKILL.md 规范一把梭：位置、YAML、触发词与冲突**
4. **04 - 指令怎么写才不飘：输入边界、步骤分解、输出模板**
5. **05 - 资源文件的价值：让输出“长得一样”，让参数“可配置”**
6. **06 - 脚本集成：把确定性交给代码，把解释性交给模型**
7. **07 - 文档处理实战：Word/Excel/PDF 的最小可用 Skill 设计**
8. **08 - API 集成实战：可控、可审计、不泄露的最小安全规范**
9. **09 - 团队共享：把 Skills 变成可版本化的“团队资产”**
10. **10 - 生产级 Skill：错误处理、性能、边界与“可回滚”**
11. **11 - 测试与调试：让 Skill 稳定复现、稳定修复**
12. **12 - 资源库运营：search → daily → briefing 的闭环**
