# 10｜生产级 Skill：错误处理、性能、边界与“可回滚”

关键词：cursor, claude code, skills, skill

当 Skill 从“自己用着爽”走到“团队依赖它”，你需要把它当成一个小产品：**要稳定、要可预期、要能在出问题时快速回滚**。

## 1）生产级的四个关键词

- **边界**：什么输入直接拒绝（例如不允许处理敏感文件/不允许任意 URL）
- **错误处理**：失败要能解释清楚，并给出下一步建议
- **性能**：大文件/批量任务要有策略（分页、限流、缓存）
- **可回滚**：改动要可追踪，必要时能恢复到上一个稳定版本

## 2）本仓库的生产级示例

看 `examples/08-production-ready/`：它体现了“资源 + 配置 + 规范”的组合思路。真正的生产级，不是某个炫酷能力，而是把每一步都做成可控的。

## 3）你应该优先补齐的“非功能需求”

如果只能选 3 件事来做生产化，建议是：

1. **输入校验**（缺参数/参数非法怎么办）
2. **输出稳定**（模板字段不可缺省）
3. **失败策略**（超时、权限不足、解析失败如何处理）

把这三件事做完，你的 Skill 就已经比大多数“演示型 Skill”更接近可用。

## 4）判断题（欢迎评论区回复“对/错”）

**判断题：** 生产级 Skill 最重要的是增加更多能力点（支持更多文件/更多接口），稳定性和错误处理可以后补。（对 / 错？）
