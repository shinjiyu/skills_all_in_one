# 12｜资源库运营：search → daily → briefing 的闭环

关键词：cursor, claude code, skills, skill

如果你想让 Skills “越用越值钱”，最后一定会走到运营：持续收集、持续整理、持续对外输出。本仓库给了一套很清晰的闭环：**搜索 → 记录 → 汇总**。

## 1）资源库的“内容形态”

`community-resources/` 里按类型维护内容：

- `blog-posts.md`：博客/文档
- `github-repositories.md`：示例仓库与工具
- `tools-and-plugins.md`：工具与插件
- `CHANGELOG.md`：更新记录（非常重要）
- `reports/`：产出的日报/简报

这套结构的核心价值是：资源不是“收藏夹”，而是“可追踪、可复用、可发布”的内容资产。

## 2）三段式 Skills：把运营流程标准化

仓库里的 `.claude/skills/` 提供了三种能力：

- `resource-search`：定义“搜什么/不搜什么”的筛选规则（只收 Skills 相关）
- `daily-report`：把每天变化写成可追踪日报
- `resource-briefing`：按周/月/时间段生成可对外发布的简报

这三段式的好处是：每一步都可独立迭代，也不会把一个 Skill 写成“万能怪物”。

## 3）权威入口要优先入库

从资源建设角度，优先级通常是：

1. 官方文档与官方示例仓库（作为规范锚点）
2. 高质量社区实践（可落地、可复用）
3. 工具链（提高效率、降低维护成本）

本仓库近期也补齐了 Claude Code 与 Cursor 的 Skills 文档入口，确保“规范有锚点”。

## 4）判断题（欢迎评论区回复“对/错”）

**判断题：** 维护资源库时，先把官方文档/官方示例作为“规范锚点”入库，通常比先大量收集零散教程更有效。（对 / 错？）
